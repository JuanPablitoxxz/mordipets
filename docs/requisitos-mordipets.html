1. INFORMACIÓN GENERAL DEL PROYECTO

1.1 Descripción del Proyecto

Mordipets es una aplicación web desarrollada para la gestión y venta de galletas para perros. La aplicación permite a los administradores gestionar inventario, insumos y pedidos, mientras que los clientes pueden explorar el catálogo de productos, realizar pedidos y gestionar sus compras.

1.2 Objetivos del Sistema

  • Digitalizar la gestión de inventario de galletas para perros
  • Facilitar el proceso de compra para los clientes
  • Automatizar la gestión de pedidos y entregas
  • Proporcionar una plataforma moderna y fácil de usar
  • Mejorar la experiencia del cliente y la eficiencia operativa

1.3 Alcance del Sistema

El sistema incluye funcionalidades para administradores (gestión completa) y clientes (compra y seguimiento), con un catálogo de 15 productos diferentes de galletas para perros, sistema de carrito de compras, gestión de pedidos y múltiples métodos de pago.

2. REQUISITOS FUNCIONALES

2.1 Gestión de Usuarios

RF-001
Registro de Clientes
El sistema debe permitir a los usuarios registrarse como clientes proporcionando información personal completa.
ALTA
Historia de Usuario:

Como cliente potencial, quiero registrarme en el sistema para poder realizar compras de galletas para mi perro.

Criterios de Aceptación:

  • El formulario debe solicitar: nombre completo, correo electrónico, número de teléfono, ubicación/establecimiento y contraseña
  • Validar que el correo electrónico sea único
  • Validar que las contraseñas coincidan
  • Mostrar mensaje de confirmación al completar el registro
  • Redirigir automáticamente al panel de cliente
RF-002
Autenticación de Usuarios
El sistema debe permitir el inicio de sesión tanto para clientes como para administradores.
ALTA

Criterios de Aceptación:

  • Formulario de login con correo electrónico y contraseña
  • Checkbox para identificar usuarios administradores
  • Validación de credenciales
  • Redirección al panel correspondiente según el tipo de usuario
  • Manejo de errores de autenticación
RF-003
Cierre de Sesión
El sistema debe permitir a los usuarios cerrar sesión de forma segura.
MEDIA

Edición de Productos
El administrador debe poder modificar la información de productos existentes.
MEDIA

Eliminación de Productos

4. ESPECIFICACIONES TÉCNICAS

4.1 Tecnologías Utilizadas

  • HTML5: Estructura semántica y accesible
  • CSS3: Estilos modernos con Flexbox y Grid
  • JavaScript ES6+: Funcionalidad interactiva
  • Font Awesome: Iconografía
  • Google Fonts: Tipografía Poppins
  • LocalStorage: Persistencia de datos

4.2 Arquitectura del Sistema

  • Frontend: Aplicación web de una sola página (SPA)
  • Almacenamiento: LocalStorage del navegador
  • Navegación: Sistema de pestañas y modales
  • Estado: Gestión de estado con JavaScript vanilla

4.3 Estructura de Datos

  • Productos: ID, código, nombre, descripción, precio, stock, peso
  • Insumos: ID, nombre, tipo, cantidad, unidad
  • Pedidos: ID, cliente, items, total, método de pago, estado, fecha
  • Usuarios: Nombre, email, teléfono, ubicación, tipo (admin/cliente)

4.4 Características de Diseño

  • Colores: Naranja (#ff6b35) y Turquesa (#20c997)
  • Tipografía: Poppins (Google Fonts)
  • Iconos: Font Awesome 6.0
  • Responsive: Mobile-first design
  • Animaciones: CSS transitions y transforms

5. CASOS DE USO PRINCIPALES

5.1 Casos de Uso del Administrador

  • Gestionar Inventario: Añadir, editar y eliminar productos
  • Controlar Insumos: Administrar ingredientes y materias primas
  • Procesar Pedidos: Ver, confirmar y marcar pedidos como entregados
  • Monitorear Stock: Revisar niveles de inventario

5.2 Casos de Uso del Cliente

  • Explorar Catálogo: Ver productos disponibles y buscar por nombre
  • Realizar Compras: Agregar productos al carrito y realizar pedidos
  • Gestionar Pedidos: Ver el estado de sus pedidos
  • Elegir Método de Pago: Pagar online o contraentrega

6. CRITERIOS DE ACEPTACIÓN GENERALES

6.1 Funcionalidad

  • Todos los formularios deben validar datos de entrada
  • Los mensajes de error deben ser claros y útiles
  • Las acciones del usuario deben tener feedback inmediato
  • Los datos deben persistir entre sesiones

6.2 Usabilidad

  • La interfaz debe ser intuitiva para usuarios sin experiencia
  • La navegación debe ser consistente en toda la aplicación
  • Los botones y enlaces deben ser claramente identificables
  • La aplicación debe funcionar correctamente en dispositivos móviles

6.3 Rendimiento

  • La aplicación debe cargar en menos de 3 segundos
  • Las búsquedas deben ser instantáneas
  • Las animaciones deben ser suaves (60fps)
  • No debe haber bloqueos de la interfaz
